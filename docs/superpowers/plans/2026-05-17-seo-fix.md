> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

## Tâche 2 : Context processor canonical URL global

**Fichiers :**
- Créer : `base/context_processors.py`
- Modifier : `main/settings.py`

**Pourquoi :** Actuellement, `canonical_url` n'est injecté que dans la homepage. Toutes les autres pages n'ont pas de balise `<link rel="canonical">`, ce qui peut créer du contenu dupliqué et des signaux contradictoires pour Google.

- [ ] **Étape 1 : Créer le context processor**

Créer le fichier `base/context_processors.py` :

```python
def canonical_url(request):
    return {
        'canonical_url': request.build_absolute_uri(request.path),
    }
```

- [ ] **Étape 2 : L'enregistrer dans settings.py**

Dans `main/settings.py`, trouver la clé `TEMPLATES` → `OPTIONS` → `context_processors` et ajouter :

```python
'base.context_processors.canonical_url',
```

La liste complète ressemblera à :

```python
'context_processors': [
    'django.template.context_processors.debug',
    'django.template.context_processors.request',
    'django.contrib.auth.context_processors.auth',
    'django.contrib.messages.context_processors.messages',
    'base.context_processors.canonical_url',  # ← ajouter ici
],
```

- [ ] **Étape 3 : Supprimer le `canonical_url` redondant dans base/views.py**

Dans `base/views.py`, la view `home()` passe déjà `canonical_url` manuellement. Le context processor le fait maintenant automatiquement. Retirer la ligne redondante :

```python
# AVANT
def home(request):
    return render(request, 'home/index.html', {
        'faqs': FAQS,
        'canonical_url': request.build_absolute_uri('/'),  # ← retirer cette ligne
    })

# APRÈS
def home(request):
    return render(request, 'home/index.html', {
        'faqs': FAQS,
    })
```

- [ ] **Étape 4 : Vérifier que le canonical apparaît sur /tarifs/**

```bash
python manage.py runserver
curl -s http://localhost:8000/tarifs/ | grep canonical
```

Résultat attendu :
```html
<link rel="canonical" href="http://localhost:8000/tarifs/">
```

- [ ] **Étape 5 : Commit**

```bash

git commit -m "seo: add global canonical URL context processor for all pages"
```

from django.shortcuts import render
from django.http import Http404
from base.blog_data import ARTICLES, get_article


def blog_index(request):
    return render(request, 'home/blog_index.html', {
        'articles': ARTICLES,
    })


def blog_detail(request, slug):
    article = get_article(slug)
    if not article:
        raise Http404
    articles_recents = [a for a in ARTICLES if a['slug'] != slug][:3]
    return render(request, 'home/blog_article.html', {
        'article': article,
        'articles_recents': articles_recents,
    })

- [ ] **Étape 1 : Créer un compte Google Search Console**

Aller sur https://search.google.com/search-console — se connecter avec le compte Google du propriétaire du site.

- [ ] **Étape 2 : Ajouter la propriété openfood.site**

Choisir "Propriété de domaine" → entrer `openfood.site` → suivre les instructions de vérification DNS (ajouter un enregistrement TXT chez votre hébergeur de domaine).

- [ ] **Étape 3 : Soumettre le sitemap**

Dans la Search Console → "Sitemaps" → entrer `https://openfood.site/sitemap.xml` → "Envoyer".

- [ ] **Étape 4 : Demander l'indexation des pages principales**

Dans la Search Console → "Inspection d'URL" → entrer chaque URL → "Demander l'indexation" :
1. `https://openfood.site/`
2. `https://openfood.site/tarifs/`
3. `https://openfood.site/benin/`
4. `https://openfood.site/cote-divoire/`
5. `https://openfood.site/senegal/`
6. `https://openfood.site/mali/`
